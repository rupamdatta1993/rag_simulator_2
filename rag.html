This simulator shows how Retrieval-Augmented Generation (RAG) works step-by-step: documents → chunking → embeddings → vector search → retrieval → AI answer generation.

RAG Pipeline

Interactive Simulation

1. Documents

Raw knowledge source. PDFs, websites, notes, databases, manuals, etc.

→

2. Chunking

Large documents are broken into smaller chunks so AI can search efficiently.

→

3. Embeddings

Text chunks become vectors (numbers) representing meaning and context.

→

4. Vector Search

User query is converted into embedding and matched with similar vectors.

→

5. AI Response

Retrieved context is sent to LLM which generates the final answer.
